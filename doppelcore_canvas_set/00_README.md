# DoppelCore Canvas Set

**Date and Time:** 2026-04-18 07:44:18

## Purpose

This canvas set defines **DoppelCore** as the machine-truth kernel for a code-mirror system. It is written for the current ForgeCommand direction where the operator is a **single-owner authority** and the primary goal is **maximum code-mirror fidelity**, not legacy protocol preservation for its own sake.

## Core Thesis

DoppelCore is the inner canonical layer.

- **Code** is reality.
- **DoppelCore** is the nearest governed machine-readable mirror of that reality.
- **Registry** is the outer control and enforcement shell.
- **Human-readable documents** are deterministic rendered products, not the authority layer.

## Relationship to existing ForgeCommand architecture

This set assumes the current ForgeCommand backend already owns a real registry-governance spine with discovery, intake, designation, compliance, persistence, events, remediation evidence, and verification evidence, and that current Registry compliance remains bounded, local-only, and documentation-governance-first. It also assumes Cortex already exists as a bounded extraction-oriented subsystem and should remain distinct from canonical truth unless explicitly re-chartered.
